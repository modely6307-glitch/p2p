# Gull - 產品更動紀錄 (Changelog)

所有對 Gull 平台的技術與產品更動都會記錄於此。

---

## [1.3.8] - 2026-03-10
### Added
- **動態 URL 管理系統 (Dynamic URL Utility)**: 
    - 新增 `getURL` 工具函式，集中管理平台跳轉邏輯。
    - 解決 Google 登入與綠界 (ECPay) 回傳連結在開發環境 (`localhost`) 與生產環境切換時的痛點。
    - 支援透過 `NEXT_PUBLIC_SITE_URL` 與 `NEXT_PUBLIC_VERCEL_URL` 環境變數精確控制回傳路徑，並在未設定時自動回退至當前瀏覽器 origin。

## [1.3.7] - 2026-03-10
### Added
- **爭議裁決回溯機制 (Dispute State Rewind)**:
    - 管理員在裁處爭議時，現在可以選擇將訂單「還原」至進入爭議前的快照狀態。
    - 資料庫層級新增 `previous_status` 欄位，自動捕捉 `DISPUTE` 發生前的進度（如 `ESCROWED`, `BOUGHT`），有效解決因誤會或文件不齊導致訂單卡死的僵局。
- **後端強制安全校驗 (Backend Security Enforcement)**:
    - 全面排除前端直接操作狀態機與統計數據的風險。
    - 核心權限邏輯遷移至 Server Actions，並實施嚴格的狀態切換校驗，確保統計數據（累計金額、訂單數）受後端鎖定保護。

## [1.3.6] - 2026-03-10
### Added
- **群組接單履約儀表板 (Group Order Fulfillment Dashboard)**:
    - 當旅人一次接起多筆相同願望的訂單（主單＋跟單）時，在訂單詳情頁面 (`/orders/[id]`) 新增「群組進度總覽」儀表板。
    - 旅人可透過儀表板輕鬆切換不同買家的操作視角，包含獨立的價格資訊、收件門市與出貨進度。
- **個別化評價系統 (Individual Rating System)**:
    - 針對群組接單場景，實現「每一張子訂單」皆可獨立進行正負評。
    - 資料庫新增 `rated_by_buyer` 與 `rated_by_traveler` 欄位，確保對不同買家的評價互不干擾。
- **分頁式群組聊天 (Tabbed Group Chat)**:
    - 旅人端聊天室現在支援分頁切換，可直接在同一個頁面與不同買家（買家 A、B、C）進行一對一對話，大幅提升跨單溝通效率。
- **證據同步與批次出貨 (Sync & Batch Actions)**:
    - 支援「一鍵同步採購證明」至同群組的其他訂單，減少重複上傳。
    - 提供「清單式批次填寫單號」介面，讓旅人能一次完成多筆訂單的發貨作業。

### Changed
- **動態 UI 上下文重構 (Dynamic UI Context Refactoring)**:
    - 整個訂單詳情頁從「基於網址 ID 的靜態渲染」重構為「基於當前選取對象 (currentViewOrder) 的動態渲染」，完美解決旅人切換買家分頁後 UI 狀態不一致的問題。


## [1.3.5] - 2026-03-09
### Added
- **訪客瀏覽機制 (Guest Viewing)**:
    - 解鎖訂單詳情頁面 (`/orders/[id]`) 的登入限制，訪客與未登入使用者現在可以直接查看願望內容與進度資訊，大幅提升平台分享性。
- **表單防呆驗證 (Explicit Form Validation)**:
    - 在「發布願望」表單中加入主動式驗證提醒 (Alert)，當必填欄位（如品名、預期到貨日、收件資訊）缺失時，系統會立即給予明確中文提示，避免原本隱含的 HTML 驗證導致按鈕「沒反應」的困擾。

### Changed
- **驗證架構優化 (Auth Refactor - Stability)**:
    - 徹底移除 `AuthContext` 中依賴慢速網路請求的 `getUser()` 初始化方式。
    - 全面改採 `getSession()` (本地快取讀取) 搭配 `onAuthStateChange` (自動 Token 刷新) 的高效模式，徹底解決原本頁面在軟刷新 (F5) 時偶爾卡住 3~5 秒的技術債。
- **跟單邏輯收斂 (Follow Order Refinement)**:
    - 為確保集體許願的成功率，將「我也要許這個願望 (跟單)」功能限定於採用「優先刊登 (立即託管)」模式的廣告，一般模式廣告則維持單一對接路徑。

### Fixed
- **多重加載問題 (Duplicate Fetching)**: 修正發布頁面與通知系統中重複抓取 Profile 的問題，統一由全局 Context 提供資料流。
- **翻譯與 UI 狀態**: 補全「已通知付款」相關翻譯 Key，並修復取消跟單後的操作導航邏輯（直接返回首頁）。

## [1.3.0] - 2026-03-05
### Added
- **雙向日期區間過濾 (Advanced Date Filtering)**:
    - 行程廣場搜尋由單一日期 (`tripDate`) 升級為區間範圍 (`departureDate` 與 `returnDate`)。
    - 首頁代購引導卡片升級，支援填寫「出發日」與「回程日」，精準查找符合行程重疊區間的願望。
    - 廣場列表邏輯自動過濾掉出發日之前的已過期願望，確保代購只看到可服務的有效訂單。
- **部分支付與訂金系統 (Partial Payment & Deposit)**:
    - 系統引入 `deposit_threshold_days` (預設 30 天) 設定，管理員可於後台動態配置。
    - 發布願望時，若「預期寄出日」距離今日超過此天數門檻，網頁將自動浮現「預付訂金」選項。
    - 支援自選訂金比例 (30%, 50%, 70%)，並即時運算「當期應付訂金」與「剩餘尾款」，清晰顯示於訂單明細中。
- **物流整合：7-11 交貨便 (Logistics Integration)**:
    - 新增買家端「取貨方式」選擇功能 (宅配到府、7-11)。
    - 成功串接綠界 7-11 電子地圖 (eMap)，買家能在彈出視窗靈活選擇或搜尋門市，並自動帶回店號 (`store_id`)、店名 (`store_name`) 與地址資訊與表單綁定。

### Fixed
- **圖片上傳相容性 (File Upload Chrome Fix)**:
    - 徹底修正因 Chrome 瀏覽器安全性升級，導致原本藏於 `<label>` 內部且設定 `display: none` 的 file input 無法開啟選擇器對話框的問題。
    - 全面重構實作方式，改用業界最佳實踐的「Opacity Hack」(利用絕對定位搭配 `opacity-0` 覆蓋於按鈕上方)，100% 確保全平台與各大瀏覽器的檔案上傳皆能正常觸發原生彈窗處理。


## [1.2.5] - 2026-02-28
### Added
- **未讀通知系統 (Notification System)**:
    - 新增 `NotificationContext`，每 15 秒輪詢用戶相關訂單狀態變更。
    - 使用 `localStorage` 存儲訂單狀態快照與已關閉通知，實現跨頁面持久化。
    - 狀態變更觸發 Toast 通知，從頂部滑入，顯示商品名稱與狀態變化 (如「待接單 → 待付款」)。
    - Toast 包含「查看訂單」快捷連結，手動關閉後不再重複彈出。
    - Dashboard tab 在有未讀通知時顯示紅色圓點計數徽章（手機底部導覽與桌面側邊欄）。

## [1.2.0] - 2026-02-28
### Added
- **下架機制 (Delisting)**:
    - 新增 `DELISTED` 訂單狀態。
    - 許願者可主動下架 `OPEN` 狀態的願望，管理員可下架 `OPEN` / `MATCHED` 狀態。
    - 下架的願望不再出現在廣場，後端阻擋對已下架願望的接單。
    - 許願者可將已下架的願望重新上架為 `OPEN`。
    - 過期願望（`expected_shipping_date` 已過且 `auto_extend = false`）自動下架。
    - Dashboard 中，下架的願望歸類至「已完成」分頁。
    - Admin 後台可見下架按鈕，分類至已完成篩選。

## [1.1.5] - 2026-02-28
### Added
- **AI 購物推薦 (AI Insights)**: 
    - 首頁加入「幫我推薦最近夯什麼」動態卡片。
    - 結合魔法棒 (Wand2) 與星空漸變視覺，提升 AI 智慧感。

## [1.1.0] - 2026-02-28
### Added
- **品牌重塑 (Branding)**: 平台正式更名為 **Gull**，副標為「Gull 購快，世界沒界限」。
- **全新 Landing Page**: 
    - 實現了「分流引導」設計，區分許願者與旅遊者路徑。
    - 視覺全面升級，符合現代 Premium 網頁審美，加入動態模糊背景與進場動畫。
    - 在手機解析度下優化標題顯示。
- **時間維度管理 (Date Logic)**:
    - 許願流程新增「預期寄出日」與「自動延續一週」開關。
    - 廣場 (`/market`) 預設按寄出日期排序，最快到期的排前面。
    - 旅遊者可以在首頁輸入旅遊日期直接導航至帶有日期篩選的廣場。
- **訂單詳情優化**: 在代購接單前的「待接單」狀態，極簡化操作介面，僅保留「接下此單」按鈕。

### Changed
- **路由調整**: 將原本的 `/` 改為 `/market`，而 `/` 現在做為 Landing Page。
- **翻譯優化**: 修正了部分 P2P 相關過時文案，統一改為 Gull 品牌風格。

## [1.0.5] - 2026-02-27
### Added
- **平台服務費 (Platform Fee)**: 
    - 實現買家與代購兩端的費率計算。
    - 買家服務費門檻化（1000 TWD 作為分界點）。
- **採購證明要求 (Proof of Purchase)**: 
    - 許願者發布時可勾選：商品採購照（必填）、收據副本、型號比對。
    - 代購在 `ESCROWED` 狀態必須上傳照片後才能點擊「完成採買」。
- **匯率顯示**: 計算公式內所有金額均加上台幣 (NT$) 或原幣別單位，提高財務透明度。

### Fixed
- **翻譯修正**: 修正了 `ESCROWED` 狀態在中文翻譯中未對應的問題。
- **隱私控制**: 買家地址僅在付款 (`ESCROWED`) 後對接單代購揭露。
