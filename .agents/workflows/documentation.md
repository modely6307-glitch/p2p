---
description: 如何記錄與更新 Gull 項目文檔
---

每次完成一個功能模組、視覺修複或業務邏輯變更後，請執行以下步驟：

1. **更新 CHANGELOG.MD**:
   - 在 `CHANGELOG.MD` 中新增或更新 `[1.x.x]` 版本。
   - 使用 `Added` (新增功能), `Changed` (更動/優化), `Fixed` (修復) 等標籤。
   - 確保日期與當前變更內容準確對應。

2. **更新 PRD.MD**:
   - 若此次變更涉及「核心業務邏輯」、「新數據庫欄位」或「新的用戶流程」，請同步更新 `PRD.MD` 中的對應章節。
   - 確保 `PRD.MD` 的最後更新日期與版本號一致。

3. **版本號規則**:
   - 視覺微調或文本修正：微版本號 (1.1.x)。
   - 新功能模块：小版本號 (1.x.0)。
   - 重大項目方向調整：大版本號 (x.0.0)。

// turbo
4. 更新完畢後，向用戶報告更新了哪些文檔。
